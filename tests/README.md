# Тестирование проекта Skills

## Структура тестов

```
tests/
├── setup.js              # Настройка тестовой среды
├── config.test.js        # Тесты конфигурации тем
├── progress.test.js      # Тесты модуля прогресса
├── main.test.js          # Тесты основной навигации
├── integration.test.js   # Интеграционные тесты
└── README.md            # Документация
```

## Установка

1. Установите зависимости:
```bash
npm install
```

## Запуск тестов

```bash
# Запустить все тесты
npm test

# Запустить тесты в режиме наблюдения
npm run test:watch

# Запустить тесты с покрытием кода
npm run test:coverage
```

## Покрытие тестами

- **config.test.js** - Тестирование конфигурации тем, валидация данных, санитизация HTML
- **progress.test.js** - Тестирование сохранения прогресса, localStorage, статистики
- **main.test.js** - Тестирование навигации, фильтрации, обработки событий
- **integration.test.js** - Тестирование взаимодействия компонентов

## Что тестируется

### Модуль config.js
- ✅ Структура данных TOPICS_CONFIG
- ✅ Валидность полей тем
- ✅ Уникальность ID тем
- ✅ Валидность категорий
- ✅ Функция санитизации HTML
- ✅ Создание HTML карточек

### Модуль progress.js
- ✅ Сохранение прогресса в localStorage
- ✅ Получение изученных тем
- ✅ Отметка тем как изученных
- ✅ Проверка изученности тем
- ✅ Вычисление статистики
- ✅ Сброс прогресса
- ✅ Обработка ошибок

### Модуль main.js
- ✅ Фильтрация тем по поисковому запросу
- ✅ Фильтрация по категориям
- ✅ Навигация между страницами
- ✅ Обработка URL параметров
- ✅ История браузера

### Интеграционные тесты
- ✅ Взаимодействие модулей
- ✅ Обновление UI при изменении прогресса
- ✅ Корректность вычислений статистики

## Добавление новых тестов

При добавлении новой функциональности:

1. Добавьте unit-тесты для новой функции
2. Обновите интеграционные тесты при необходимости
3. Проверьте покрытие кода: `npm run test:coverage`
4. Убедитесь, что все тесты проходят

## Известные ограничения

- Тесты для DOM-манипуляций требуют jsdom окружения
- Некоторые функции main.js требуют рефакторинга для экспорта (тестирование через публичный API)
- Fetch API мокируется для тестирования загрузки страниц
